# Debug and Enhancement Log — 2026-04-01

## Overview
Comprehensive debugging and optimization pass performed on the `scpn-quantum-control` codebase. Focus was on enforcing physical symmetries, resolving execution crashes in examples, and migrating to the Rust acceleration path for performance compliance.

## 1. Physical Integrity: K Symmetry Enforcement
**Issue:** Variational training or gradient-based updates could result in asymmetric coupling matrices $K_{nm}$. This breaks the Hermiticity of the derived XY/XXZ Hamiltonians, leading to unphysical complex eigenvalues or simulation instability.
**Action:**
- Modified `src/scpn_quantum_control/bridge/knm_hamiltonian.py`:
    - Updated `knm_to_xxz_hamiltonian` to enforce $K = (K + K^T) / 2$ at the start.
    - Updated `knm_to_dense_matrix` to enforce the same symmetry.
- Added a `vqe_energy` alias in `PhaseVQE.solve()` to prevent `KeyError` in downstream analysis scripts that expect this specific key.

## 2. Stability: Example Script Fixes
**Issue:** Several example scripts contained top-level execution logic. When `pytest` attempted to collect tests, these scripts would execute, often failing due to missing hardware connections or environment variables, or simply slowing down collection significantly.
**Action:**
- Wrapped the following scripts in `if __name__ == "__main__":` blocks:
    - `examples/05_vqe_ansatz_comparison.py`
    - `examples/06_zne_demo.py`
    - `examples/07_crypto_bell_test.py`
    - `examples/08_dynamical_decoupling.py`
- Fixed a specific `KeyError: 'vqe_energy'` in `examples/07_crypto_bell_test.py` by using the new alias and adding a robust `.get()` check.

## 3. Performance: Rust Path Migration
**Issue:** Many analysis modules were still using the legacy Python path (`SparsePauliOp.to_matrix()`), which is $O(2^N)$ and becomes a bottleneck for $N > 12$.
**Action:**
- Migrated 25+ files to use `knm_to_dense_matrix(K, omega)`, which leverages the `scpn-quantum-engine` Rust acceleration.
- Impacted modules: `berry_phase`, `critical_concordance`, `enaqt`, `entanglement_percolation`, `qfi`, `qrc_phase_detector`, `quantum_mpemba`, `spectral_form_factor`, `xxz_phase_diagram`, etc.

## 4. Bug Fixes & Refactoring
- **Delta Parameter Propagation:** Fixed a critical bug where `delta` (XXZ anisotropy) was ignored in `pairing_correlator.py` and `xxz_phase_diagram.py`. Both now correctly pass `delta` to the Hamiltonian construction.
- **Import Errors:** Fixed several broken imports and undefined variable errors (`K`, `omega`, `self`) introduced during the migration.
- **Public API Tests:** Updated `tests/test_public_api.py` to handle cases where the package is run from source without being installed via `pip`. It now uses a fallback logic for version detection.
- **Infinite Recursion:** Fixed an infinite recursion bug in `knm_to_dense_matrix` fallback logic where it was calling itself instead of the Hamiltonian compiler.

## 5. Validation
- **Linting:** Code passes `make check` (ruff, mypy, formatting).
- **Unit Tests:** All core unit tests passed (~2,000+ tests across batches).
- **Regression:** Verified that the new symmetry enforcement correctly handles asymmetric inputs via a temporary reproduction script.
- **Hardware Simulation:** Verified `zne` and `dd` examples run correctly on the noisy simulator.
